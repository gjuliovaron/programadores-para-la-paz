Registro de ejecución del servidor

1. ¿Qué ocurre cuando ejecutas node server.js?

Al ejecutar el comando node server.js en la terminal, Node.js lee el archivo server.js, carga el framework Express, configura las rutas definidas y pone el servidor a escuchar en el puerto 3000. El proceso queda activo en la terminal, esperando solicitudes entrantes.

2. ¿Qué mensaje aparece en la terminal?

Aparece el mensaje: "Servidor ejecutándose en puerto 3000". Esto confirma que el servidor inició correctamente y está listo para recibir peticiones.

3. ¿Qué sucede cuando visitas la ruta /estado?

Al visitar http://localhost:3000/estado en el navegador, el servidor responde con un objeto JSON que indica el estado del servicio:

{
  "estado": "Servidor funcionando",
  "servicio": "API comunitaria"
}

Esto permite verificar que la API está activa y operativa.
